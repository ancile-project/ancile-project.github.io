Ancile

Campus Infrastucture
for Data-Driven Planning

Ancile is a framework that is designed to support the creation of microscale, privacy-concious, data-driven planning applications on the Cornell Tech Campus. Ancile will dramatically lower the bar for building advanced planning applications while giving individuals fine-grained visibility and control over how their personal data is used and shared.

Privacy Preservation and Management

Ancile provides high level controls to users so that they can manage how their data is being used to help develop the campus.

Small Data Core API

One of the key aspects of Ancile is utilizing personal user data in a privacy-preserving way. The Small Data Core API is Ancile’s arbiter for personal user data, providing stronger protections on access to a user's personal data.

Small Data Core provides additional scoping to existing data APIs, which allows users have a greater idea of what personal data is being used, and for apps to have a greater ability to request only the data that they absolutely require.

A simple example of this is an application that only needs access to events on your calendar that take place between certain times or that occur in a specific place. Currently, you’d need to provide access to your entire Google calendar to the app, however Small Data Core can enforce this additional filtering and communicate that to the user.

The following shows the general architecture of Small Data Core.

An app that uses Small Data Core will go through the following process to retrieve a user’s data:

  • The app registers with Small Data Core
  • The app tells Core what data it would like to request from users
  • The app redirects a user to authenticate with Core
  • The user is given details on what data is being requested
  • Upon approval, the data is accessible by the app
  • The user can revoke permission at any time
